# Launch Strategy Documentation

This folder contains all documentation related to the March 15th launch strategy and AI agent workflows for the Evega project.

## Files in this folder:

1. **`CTO_AGENT_STRATEGY.md`** - Comprehensive guide on using AI agents (Cursor, GitHub Copilot) to accelerate development toward the March 15th launch deadline.

2. **`LAUNCH_ROADMAP.md`** - Detailed 4-week roadmap breaking down the launch preparation into phases with specific tasks and milestones.

3. **`GITHUB_WORKFLOW.md`** - Guide for using the PR template and GitHub workflow for the launch process.

4. **`MULTI_AGENT_SETUP.md`** - **NEW** - Step-by-step guide to create CTO, Architect, and QA agents with complete setup instructions.

5. **`AGENT_SWITCHING.md`** - **NEW** - Guide on how to switch between different agents in Cursor.

6. **`AGENT_COLLABORATION.md`** - **NEW** - Workflow for how agents collaborate to implement features.

## Quick Start

### For Multi-Agent System Setup:
1. **Read `MULTI_AGENT_SETUP.md`** - Complete step-by-step guide to set up CTO, Architect, and QA agents
2. **Follow the 10 steps** in MULTI_AGENT_SETUP.md to configure all agents
3. **Use `AGENT_SWITCHING.md`** to learn how to switch between agents
4. **Review `AGENT_COLLABORATION.md`** to understand how agents work together

### For Launch Planning:
1. **Read the Launch Roadmap** to understand the timeline and phases
2. **Review the CTO Agent Strategy** to learn how to use AI agents effectively
3. **Follow the daily workflow** outlined in the strategy document
4. **Track progress** using the daily progress script: `./scripts/daily-progress.sh`
