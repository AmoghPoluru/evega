# AI Agent - GitHub Architecture Overview

> **Purpose**: This document explains where everything happens in GitHub and how the AI agent system integrates with GitHub features.

---

## GitHub Components Used

### 1. **Repository (Your Codebase)**
**Location**: Your main repository (e.g., `your-org/evega`)

**What's Stored Here:**
- **Agent Code**: `.github/agents/` folder
  - All agent scripts, tools, and logic
  - Lives in your repo, version controlled
  - Example: `.github/agents/agent.js`, `.github/agents/tools/*.js`

- **Workflow Definition**: `.github/workflows/ai-issue-fix.yml`
  - GitHub Actions workflow file
  - Defines when and how the agent runs
  - Lives in your repo, version controlled

- **Agent Rules**: `AGENTS.md` (repo root)
  - Instructions for the AI agent
  - Lives in your repo, version controlled

**Visual Structure:**
```
your-repo/
├── .github/
│   ├── workflows/
│   │   └── ai-issue-fix.yml          ← Workflow definition
│   └── agents/
│       ├── agent.js                   ← Main agent script
│       ├── tools/                     ← Agent tools
│       ├── core/                      ← Core logic
│       └── safety/                    ← Safety checks
├── AGENTS.md                          ← Agent rules
└── (rest of your code)
```

---

### 2. **GitHub Actions (Cloud Runners)**
**Location**: GitHub's cloud infrastructure (not your local machine)

**What Happens Here:**
- **Workflow Execution**: When triggered, GitHub spins up a virtual machine (runner)
- **Environment**: Ubuntu Linux (or Windows/macOS if configured)
- **Duration**: Runs for up to 20 minutes (configurable)
- **Access**: Has access to your repo via `GITHUB_TOKEN`

**Workflow Flow:**
```
1. Issue gets `ai-fix` label
   ↓
2. GitHub Actions detects trigger
   ↓
3. Spins up Ubuntu runner (cloud VM)
   ↓
4. Checks out your repo code
   ↓
5. Runs agent script (Node.js)
   ↓
6. Agent makes changes, commits, creates PR
   ↓
7. Runner shuts down
```

**Where to See It:**
- Go to your repo → **Actions** tab
- See all workflow runs
- Click a run to see logs
- Logs show what the agent did step-by-step

---

### 3. **GitHub Issues**
**Location**: Your repo → **Issues** tab

**What's Used:**
- **Issue Creation**: You (or team) create issues describing bugs/features
- **Labels**: Applied to issues (`ai-fix`, `ai-in-progress`, etc.)
- **Comments**: Agent posts comments with results
- **Issue Number**: Used to create branch names and link PRs

**Example Issue:**
```
Issue #123: Fix cart total calculation bug
Labels: [bug] [ai-fix]
Body: "The cart total is incorrect when items are added..."
```

**Agent Interaction:**
- Reads issue title, body, labels
- Posts comments with progress/results
- Updates labels (`ai-in-progress`, `ai-blocked`, etc.)

---

### 4. **GitHub Branches**
**Location**: Your repo → **Code** tab → **Branches**

**What's Created:**
- **Agent Branches**: `ai/issue-123-fix-cart-bug`
- **Naming**: `ai/issue-<number>-<slug>`
- **Purpose**: Isolated branch for agent's changes
- **Base**: Branches from `main` (or your default branch)

**Branch Lifecycle:**
```
1. Agent creates branch: ai/issue-123-fix-cart-bug
   ↓
2. Agent commits changes to this branch
   ↓
3. Agent pushes branch to GitHub
   ↓
4. Agent creates PR from this branch
   ↓
5. After PR merge: Branch can be deleted
```

**Where to See:**
- Repo → **Code** → **Branches**
- See all branches, including agent-created ones
- Can view branch commits, files changed

---

### 5. **GitHub Pull Requests**
**Location**: Your repo → **Pull requests** tab

**What's Created:**
- **PR Title**: `Fix #123: Fix cart total calculation bug`
- **PR Description**: Generated by agent (summary, changes, how to test)
- **PR Status**: Draft (if tests fail) or Ready (if all pass)
- **PR Labels**: `ai-draft` (if draft)

**PR Contents:**
- **Files Changed**: Shows diff of agent's changes
- **Commits**: Shows commit history
- **Checks**: Shows CI/CD status (lint, test, build)
- **Comments**: Can have discussion, reviews

**PR Lifecycle:**
```
1. Agent creates PR
   ↓
2. PR appears in "Pull requests" tab
   ↓
3. Team reviews PR
   ↓
4. If approved: Merge PR
   ↓
5. Branch deleted (optional, auto-cleanup)
```

---

### 6. **GitHub Labels**
**Location**: Your repo → **Issues** → **Labels** (settings)

**What's Created:**
- `ai-fix` - Trigger label (green)
- `ai-in-progress` - Agent working (yellow)
- `ai-blocked` - Agent stuck/failed (red)
- `ai-draft` - PR is draft (gray)
- `ai-allow-sensitive` - Allow high-risk changes (red)
- `ai-no` - Opt-out label (black)

**Where They Appear:**
- On issues (when applied)
- In issue lists (filterable)
- In PRs (if PR has labels)

**How They're Used:**
- **Trigger**: Adding `ai-fix` label starts the agent
- **Status**: `ai-in-progress` shows agent is working
- **Result**: `ai-blocked` or `ai-draft` shows outcome

---

### 7. **GitHub Secrets**
**Location**: Your repo → **Settings** → **Secrets and variables** → **Actions**

**What's Stored:**
- `OPENAI_API_KEY` - Your OpenAI API key (or Anthropic, etc.)
- `GITHUB_TOKEN` - Automatically provided by GitHub (no setup needed)

**Security:**
- **Encrypted**: Secrets are encrypted at rest
- **Masked**: Never shown in logs (automatically masked)
- **Scoped**: Only accessible in GitHub Actions workflows
- **Access**: Only repo admins can view/edit secrets

**How They're Used:**
- Workflow reads secrets as environment variables
- Agent script accesses via `process.env.OPENAI_API_KEY`
- Never committed to code

## Where to Find Everything

### **Agent Code & Configuration**
- **Location**: In your repo, under `.github/agents/`
- **View**: Repo → Code tab → Navigate to `.github/agents/`
- **Edit**: Clone repo, edit files, commit and push

### **Workflow Definition**
- **Location**: `.github/workflows/ai-issue-fix.yml`
- **View**: Repo → Code tab → `.github/workflows/ai-issue-fix.yml`
- **Edit**: Same as above

### **Workflow Runs (Execution History)**
- **Location**: Repo → **Actions** tab
- **View**: See all workflow runs, click to see logs
- **Logs**: Show what agent did, any errors

### **Issues**
- **Location**: Repo → **Issues** tab
- **View**: All issues, filter by labels
- **Create**: Click "New issue", add `ai-fix` label

### **Pull Requests**
- **Location**: Repo → **Pull requests** tab
- **View**: All PRs, filter by author, labels
- **Review**: Click PR to see changes, approve/request changes

### **Branches**
- **Location**: Repo → **Code** tab → **Branches** dropdown
- **View**: All branches, including agent-created ones
- **Delete**: After PR merge, can delete branch

### **Labels**
- **Location**: Repo → **Issues** → **Labels** (or Settings → Labels)
- **View**: All labels, their colors, descriptions
- **Create**: Click "New label", set name, color, description

### **Secrets**
- **Location**: Repo → **Settings** → **Secrets and variables** → **Actions**
- **View**: Secret names (values are hidden)
- **Create**: Click "New repository secret"

## Key Points

1. **Everything is in ONE repository**
   - Agent code lives in your repo
   - Workflow runs in GitHub's cloud
   - Issues, PRs, branches are all in the same repo

2. **Agent runs in GitHub's cloud, not locally**
   - GitHub Actions provides the compute (Ubuntu VMs)
   - You don't need to run anything on your machine
   - Agent has access to your repo via GitHub API

3. **All interactions happen through GitHub UI**
   - Create issue → Add label → Agent runs → PR appears
   - You can see everything in GitHub's web interface
   - No command-line interaction needed (for users)

4. **Agent code is version controlled**
   - Agent scripts are in your repo
   - Changes to agent code go through normal PR process
   - You can improve the agent over time

5. **Security is built-in**
   - Secrets are encrypted and masked
   - Agent only has permissions you grant
   - All actions are logged in Actions tab
