# Stripe Connect Testing Guide

This guide covers how to test the Stripe Connect implementation, including setting up webhooks for local development.

## Prerequisites

1. **Stripe Account**: You need a Stripe account (test mode is fine)
2. **Stripe CLI**: Install for local webhook testing
3. **Environment Variables**: Configure your `.env.local` file

---

## Step 1: Install Stripe CLI

### macOS
```bash
brew install stripe/stripe-cli/stripe
```

### Linux/Windows
Download from: https://stripe.com/docs/stripe-cli

### Verify Installation
```bash
stripe --version
```

---

## Step 2: Login to Stripe CLI

```bash
stripe login
```

This will open your browser to authenticate with Stripe.

---

## Step 3: Configure Environment Variables

Add these to your `.env.local`:

```env
# Existing Stripe keys
STRIPE_SECRET_KEY=sk_test_... (your test secret key)
STRIPE_PUBLISHABLE_KEY=pk_test_... (your test publishable key)
STRIPE_WEBHOOK_SECRET=whsec_... (will be generated by Stripe CLI)

# Stripe Connect (optional for basic testing)
STRIPE_CONNECT_CLIENT_ID=ca_... (only needed for OAuth flow)
STRIPE_PLATFORM_ACCOUNT_ID=acct_... (optional)

# App URL
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

---

## Step 4: Start Local Webhook Listener

In a **separate terminal**, run:

```bash
stripe listen --forward-to localhost:3000/api/stripe/webhook
```

This will:
- Forward Stripe webhook events to your local server
- Display a webhook signing secret (starts with `whsec_`)
- Show all incoming webhook events in real-time

**Copy the webhook secret** and add it to your `.env.local`:

```env
STRIPE_WEBHOOK_SECRET=whsec_... (from Stripe CLI output)
```

**Important**: Restart your Next.js dev server after adding the webhook secret!

---

## Step 5: Start Your Development Server

In your main terminal:

```bash
cd evega
npm run dev
```

Your app should be running at `http://localhost:3000`

## Step 6: Test Vendor Stripe Account Creation

### 6.1 Create/Login as Vendor

1. Go to `http://localhost:3000/sign-in`
2. Sign in as a vendor (or create a vendor account)
3. Navigate to `http://localhost:3000/vendor/dashboard`

### 6.2 Connect Stripe Account

1. Click "Connect Stripe Account" or go to `http://localhost:3000/vendor/stripe-onboarding`
2. Click "Connect Stripe Account" button
3. You should be redirected to Stripe's onboarding page (test mode)

### 6.3 Complete Stripe Onboarding (Test Mode)

In Stripe's test onboarding:
- Use test data (any email, business name, etc.)
- For bank account, use Stripe test account numbers:
  - **Account number**: `000123456789`
  - **Routing number**: `110000000`
- Complete all required fields
- Submit the form

### 6.4 Verify Account Creation

1. You should be redirected back to `/vendor/stripe-onboarding?success=true`
2. Check the vendor dashboard - it should show "Active" status
3. Check Stripe CLI terminal - you should see `account.updated` webhook events

## Step 7: Test Payment Flow with Stripe Connect

### 7.1 Create a Test Product

1. Go to `http://localhost:3000/vendor/products/new`
2. Create a product with a price (e.g., $10.00)
3. Save the product

### 7.2 Add Product to Cart (as Customer)

1. Sign in as a different user (customer account)
2. Browse to the product page
3. Add product to cart
4. Go to checkout

### 7.3 Complete Test Payment

1. On checkout page, click "Proceed to Payment"
2. Use Stripe test card: `4242 4242 4242 4242`
3. Use any future expiry date (e.g., `12/34`)
4. Use any CVC (e.g., `123`)
5. Use any ZIP code (e.g., `12345`)
6. Complete the payment

### 7.4 Verify Payment Processing

**Check Stripe CLI terminal** - you should see:
- `checkout.session.completed`
- `payment_intent.succeeded`
- `transfer.created` (if using Stripe Connect)

**Check your database**:
- Order should be created with status `payment_done`
- Order should have `stripeTransferId` and `transferStatus`
- Order should have `commission` and `vendorPayout` calculated

**Check vendor dashboard**:
- Order should appear in vendor's order list
- Commission should be deducted from total

## Step 8: Test Webhook Events

### 8.1 Trigger Test Events

You can trigger test events using Stripe CLI:

```bash
# Test account.updated event
stripe trigger account.updated

# Test transfer.paid event
stripe trigger transfer.paid

# Test payment_intent.succeeded
stripe trigger payment_intent.succeeded
```

### 8.2 Verify Webhook Handling

Check your Next.js server logs and Stripe CLI output to see:
- Webhook events being received
- Orders being updated
- Vendor status being synced

## Step 9: Test Error Scenarios

### 9.1 Test Vendor Without Stripe Account

1. Create a vendor without connecting Stripe
2. Try to checkout with their product
3. Should see error: "Vendor has not connected their Stripe account"

### 9.2 Test Vendor with Pending Account

1. Create Stripe account but don't complete onboarding
2. Try to checkout
3. Should see error: "Vendor's payment account is not ready"

### 9.3 Test Transfer Failure

1. Use Stripe test mode to simulate transfer failure
2. Check that order status updates correctly
3. Check that vendor payout status reflects failure

---

## Step 10: Verify Commission Calculation

### 10.1 Check Commission Rate

1. Go to vendor settings or admin panel
2. Verify `commissionRate` is set (default: 10%)
3. Create a test order
4. Verify commission calculation:
   - `commission = (total * commissionRate) / 100`
   - `vendorPayout = total - commission`

### 10.2 Test Different Commission Rates

1. Change vendor's `commissionRate` to 15%
2. Create another test order
3. Verify new commission rate is applied

## Common Issues & Solutions

### Issue: Webhook events not received

**Solution**:
1. Make sure Stripe CLI is running: `stripe listen --forward-to localhost:3000/api/stripe/webhook`
2. Check webhook secret in `.env.local` matches CLI output
3. Restart Next.js dev server after adding webhook secret
4. Check Stripe CLI terminal for connection status

### Issue: "Vendor has not connected their Stripe account"

**Solution**:
1. Make sure vendor completed Stripe onboarding
2. Check `vendor.stripeAccountId` exists in database
3. Verify `stripeAccountStatus` is "active"

### Issue: Transfer not created

**Solution**:
1. Check that checkout uses `createCheckoutSessionWithConnect()`
2. Verify vendor's Stripe account is active
3. Check Stripe Dashboard for transfer status
4. Review webhook logs for `transfer.created` events

### Issue: Commission not calculated

**Solution**:
1. Check vendor's `commissionRate` field in database
2. Verify commission calculation in checkout procedure
3. Check order creation in webhook handler

## Production Webhook Setup

For production, you need to:

1. **Create Webhook Endpoint in Stripe Dashboard**:
   - Go to Stripe Dashboard > Developers > Webhooks
   - Click "Add endpoint"
   - URL: `https://yourdomain.com/api/stripe/webhook`
   - Select events:
     - `checkout.session.completed`
     - `payment_intent.succeeded`
     - `account.updated`
     - `transfer.created`
     - `transfer.paid`
     - `transfer.failed`
     - `transfer.canceled`

2. **Get Webhook Secret**:
   - Copy the webhook signing secret from Stripe Dashboard
   - Add to production environment variables: `STRIPE_WEBHOOK_SECRET`

3. **Test Production Webhook**:
   - Use Stripe Dashboard to send test events
   - Verify your production endpoint receives and processes them

```bash

# Trigger test events
stripe trigger checkout.session.completed
stripe trigger payment_intent.succeeded
stripe trigger account.updated
stripe trigger transfer.paid

# View webhook events
stripe events list

# View specific event
stripe events retrieve evt_...

# Test API calls
stripe accounts list
stripe transfers list
```
