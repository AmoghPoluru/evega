# CTO Agent - Ready-to-Use Prompts

## Daily Operations

### Morning Standup
```
Act as CTO. Conduct morning standup for Evega project:
- Review yesterday's commits and progress
- Assess current status against March 15th launch deadline
- Identify today's top 3 priorities
- Flag any blockers or risks
- Provide strategic recommendations
```

### End of Day Review
```
Act as CTO. Review end-of-day status:
- Summarize today's accomplishments
- Assess progress toward launch
- Identify any new risks or blockers
- Recommend tomorrow's focus areas
- Calculate launch readiness score (0-100%)
```

### Weekly Status Report
```
Act as CTO. Generate weekly status report:
- Progress against roadmap (tasks completed)
- Risk assessment (current risks and mitigation)
- Resource allocation review
- Launch readiness score
- Strategic recommendations for next week
```

## Strategic Planning

### Launch Readiness Assessment
```
Act as CTO. Assess launch readiness for March 15th:
- Review DETAILED_TASKS.md completion status
- Evaluate critical path items
- Assess bug status and severity
- Review test coverage
- Check performance metrics
- Provide go/no-go recommendation with rationale
```

### Risk Assessment
```
Act as CTO. Conduct risk assessment:
- Analyze codebase for technical risks
- Review project timeline for schedule risks
- Assess security vulnerabilities
- Evaluate performance concerns
- Prioritize risks by severity and likelihood
- Recommend mitigation strategies
```

### Priority Planning
```
Act as CTO. Review DETAILED_TASKS.md and prioritize:
- Identify critical path items for launch
- Separate must-have from nice-to-have
- Assess technical debt impact
- Balance feature development vs. bug fixes
- Create prioritized task list for next sprint
```

## Decision Making

### Go/No-Go Decision
```
Act as CTO. Make go/no-go decision for [feature/deployment]:
- Assess feature completeness
- Evaluate quality metrics
- Review test coverage
- Check for blocking issues
- Provide decision with clear rationale
```

### Resource Allocation
```
Act as CTO. Allocate resources for next sprint:
- Review current priorities
- Balance development vs. testing vs. bug fixes
- Consider launch deadline
- Recommend resource distribution
- Justify allocation decisions
```

### Technical Debt Assessment
```
Act as CTO. Assess technical debt:
- Identify technical debt items
- Evaluate impact on launch
- Prioritize debt to address
- Recommend timeline for addressing
- Balance debt vs. new features
```

## Communication

### Executive Summary
```
Act as CTO. Create executive summary:
- Current project status
- Progress toward March 15th launch
- Key metrics (completion %, test coverage, bug count)
- Top 3 risks
- Strategic recommendations
```

### Status Update
```
Act as CTO. Provide status update:
- What's been completed
- What's in progress
- What's blocked
- Launch readiness percentage
- Next steps
```
